Tutorial Cocos2d-x Part-2 (Membuat Project Baru)

Halo, para pembaca. Seperti yang sudah saya bahas pada postingan sebelumnya, saya akan membahas tentang cara membuat project baru di cocos2d-x. Mungkin untuk para pembaca yang belum membaca postingan sebelumnya bisa dilihat di:

Tutorial Cocos2d-x Untuk Pemula

Seperti pada bahasan sebelumnya, proses instalasi menggunakan perintah dalam command prompt. Untuk membuat project baru, mengcompile dan running juga akan menggunakan command prompt, jadi sebaiknya dibiasakan untuk menggunakan command prompt😀

Perintah untuk membuat project baru sebenarnya cukup sederhana, yaitu dengan perintah:

cocos new [nama game] -p [nama package] -l [bahasa pemrograman]

Contohnya adalah sebagai berikut:

cocos new GameKu -p com.perusahaanku.gameku -l cpp
cocos new GameKu -p com.perusahaanku.gameku -l lua
cocos new GameKu -p com.perusahaanku.gameku -l js 

Dalam contoh diatas, sebuah project baru dibuat menggunakan source code dari Cocos2d-x. Jika kamu ingin membuat sebuah project baru menggunakan pre-built libraries, maka kamu harus menambahkan flag -t binary. Contohnya:

cocos new GameKu -p com.perusahaanku.gameku -l cpp -t binary

Sebelum mengetikkan baris perintah diatas, sebaiknya tentukan dulu lokasi yang dijadikan tempat project baru kita. Kemudian tunggu semua proses hingga selesai.

Setelah semua proses selesai, maka kita akan mendapatkan beberapa file seperti berikut :

Setiap kita membuat project baru, maka kita akan mendapatkan struktur file seperti gambar diatas. Seperti gambar di atas, bisa diketahui bahwa cocos2d-x mensupport beberapa platform seperti android, ios, linux, tizen, windows 8.1 universal, windows 10 dan windows desktop. Berikut ini akan saya jelaskan kegunaan tiap folder-folder seperti pada gambar diatas.

Folder Classes : Di dalam folder classes ini digunakan untuk meletakkan kode-kode  dalam game yang kita buat.

Folder Resources : Di dalam folder Resources ini digunakan untuk meletakkan resource – resource yang kita butuhkan dalam membuat game, seperti gambar, musik, font, xml, dan lain sebagainya.

Folder cocos2d : Folder ini berisi source code dari cocos2d-x.

Lalu folder-folder yang lain seperti proj.android, proj.ios_mac, proj.win32, itu berisi file-file project yang bisa dibuka untuk IDE yang sesuai dengan platformnya. Misalnya folder proj.android berisi file-file project yang bisa dibuka di eclipse, folder proj.win32 berisi file-file yang bisa dibuka di Microsoft Visual Studio.

Untuk kali ini, kita akan mencoba menjalankan project kita ini menggunakan Microsoft Visual Studio. Untuk itu buka folder proj.win32, lalu pilih GameKu.sln.

Setelah anda buka, maka secara default akan ada empat file yang tersedia, yaitu AppDelegate.h, AppDelegate.cpp, HelloWorldScene.h, dan HelloWorldScene.cpp.

Jika kalian coba jalankan (Run), maka akan muncul tampilan seperti di bawah ini :

#ifndef  _APP_DELEGATE_H_
#define  _APP_DELEGATE_H_
 
#include "cocos2d.h"
 
/**
@brief    The cocos2d Application.
 
Private inheritance here hides part of interface from Director.
*/
class  AppDelegate : private cocos2d::Application
{
public:
    AppDelegate();
    virtual ~AppDelegate();
 
    virtual void initGLContextAttrs();
 
    /**
    @brief    Implement Director and Scene init code here.
    @return true    Initialize success, app continue.
    @return false   Initialize failed, app terminate.
    */
    virtual bool applicationDidFinishLaunching();
 
    /**
    @brief  Called when the application moves to the background
    @param  the pointer of the application
    */
    virtual void applicationDidEnterBackground();
 
    /**
    @brief  Called when the application reenters the foreground
    @param  the pointer of the application
    */
    virtual void applicationWillEnterForeground();
};
 
#endif // _APP_DELEGATE_H_

Ini adalah file header untuk class AppDelegate yang didefinisikan dalam AppDelegate.cpp. Didalamnya mengandung deklarasi beberapa fungsi antara lain:

  • AppDelegate() — constructor dari class AppDelegate.
  • virtual ~AppDelegate() — destructor dari class AppDelegate.
  • virtual void initGLContextAttrs() — fungsi ini bertugas untuk mengimplemen OpenGL (Open Graphics Library) context sehingga kamu bisa merender grafis 2d atau 3d ke semua platform.
  • virtual bool applicationDidFinishLaunching() — Meluncurkan dan menginisialisasi aplikasimu dan mengembalikan nilai true jika proses peluncuran berhasil dan false jika tidak.
  • virtual void applicationDidEnterBackground() — Fungsi ini dipanggil ketika aplikasi mulai berjalan di background (tidak muncul di layar). Contohnya ketika pemain mendapat panggilan telepon atau membuka aplikasi lain.
  • virtual void applicationWillEnterForeground() — Fungsi ini dipanggil ketika aplikasi kembali lagi berjalan di foreground (aplikasi muncul di layar). Contohnya ketika pemain kembali ke aplikasi setelah menerima panggilan telepon.

static cocos2d::Size designResolutionSize = cocos2d::Size(1280, 768);
 
AppDelegate::AppDelegate()
{
}
 
AppDelegate::~AppDelegate()
{

// if you want a different context, modify the value of glContextAttrs
// it will affect all platforms
void AppDelegate::initGLContextAttrs()
{
    // set OpenGL context attributes: red,green,blue,alpha,depth,stencil
    GLContextAttrs glContextAttrs = {8, 8, 8, 8, 24, 8};
 
    GLView::setGLContextAttrs(glContextAttrs);
}
 
// if you want to use the package manager to install more packages,
// don't modify or remove this function
static int register_all_packages()
{
    return 0; //flag for packages manager
}
 
bool AppDelegate::applicationDidFinishLaunching() {
    // initialize director
    auto director = Director::getInstance();
    auto glview = director->getOpenGLView();
    if(!glview) {
#if (CC_TARGET_PLATFORM == CC_PLATFORM_WIN32) || (CC_TARGET_PLATFORM == CC_PLATFORM_MAC) || (CC_TARGET_PLATFORM == CC_PLATFORM_LINUX)
        glview = GLViewImpl::createWithRect("GameKu", cocos2d::Rect(0, 0, designResolutionSize.width, designResolutionSize.height));
#else
        glview = GLViewImpl::create("GameKu");
#endif
        director->setOpenGLView(glview);
    }
 
    // turn on display FPS
    director->setDisplayStats(true);
 
    // set FPS. the default value is 1.0/60 if you don't call this
    director->setAnimationInterval(1.0f / 60);
 
    // Set the design resolution
    glview->setDesignResolutionSize(designResolutionSize.width, designResolutionSize.height, ResolutionPolicy::NO_BORDER);
 
    register_all_packages();
 
    // create a scene. it's an autorelease object
    auto scene = HelloWorld::createScene();
 
    // run
    director->runWithScene(scene);
 
    return true;
}
 
// This function will be called when the app is inactive. Note, when receiving a phone call it is invoked.
void AppDelegate::applicationDidEnterBackground() {
    Director::getInstance()->stopAnimation();
 
    // if you use SimpleAudioEngine, it must be paused
    // SimpleAudioEngine::getInstance()->pauseBackgroundMusic();
}
 
// this function will be called when the app is active again
void AppDelegate::applicationWillEnterForeground() {
    Director::getInstance()->startAnimation();
 
    // if you use SimpleAudioEngine, it must resume here
    // SimpleAudioEngine::getInstance()->resumeBackgroundMusic();
}

Ini adalah file yang mengimplementasi fungsi-fungsi yang telah dideklarasikan di AppDelegate.h antara lain:

  • AppDelegate::AppDelegate() — Membuat object AppDelegate() dengan value yang belum diinisialisasi
  • AppDelegate::~AppDelegate() — Fungsi destructor yang dipanggil ketika object AppDelegate tidak lagi dibutuhkan. Perhatikan bahwa walaupun fungsi ini kosong, bukan berarti ia tidak pernah mengeksekusi kode apapun. Compiler bisa menambahkan kode ke fungsi ini memanggil destructor dari base class dan data non-static jika diperlukan.
  • void AppDelegate::initGLContextAttrs() — Fungsi ini digunakan untuk mengatur attribut-atribut dari OpenGL seperti:
    { red, green, blue, alpha, depth, stencil }

    • Red, green, dan blue mengacu pada warna yang ditampilkan pada layarmu.
    • Alpha biasanya mengacu pada tingkat transparansi
    • Depth mengacu pada berapa jauh dari viewer dan“menuju” layar pixelmu (penulis: mohon maaf kalo bahasanya agak kacau, karena ini sumbernya berbahasa inggris dan ditranslate). Contohnya, jika kamu punya pixel A dan pixel B dengan koordinat (x, y) yang sama tapi A punya tingkat kedalaman (depth) lebih besar dari B, atau dengan kata lain pixel A ada di atas pixel B, maka kita tidak perlu menampilkan piksel B karena ia telah disembunyikan oleh piksel A. Tujuannya adalah untuk mempercepat performa dari aplikasimu.
    • Stencil memungkinkanmu untuk menyuruh OpenGL pixel yang mana yang digambar dan pixel yang mana yang disembunyikan.
  • bool AppDelegate::applicationDidFinishLaunching() — Meluncurkan aplikasimu dengan membuat dan menginisialisasi object director dan sebuah object scene.
    • Director — Object ini adalah sebuah Singleton, maksudnya programmu cuman memiliki satu object saja darinya. Object director menginisialisasi sebuah OpenGL context untuk membuat dan meng-handle jendela/layar aplikasimu, dan itu juga untuk mengontrol bagaimana/kapan untuk mengeksekusi scene.
    • Scene — Sebuah scene adalah sebuah “ruangan” yang mengandung apa saja yang bisa ditampilkan di layarmu. Pada umumnya scene pertama dalam sebuah game mengandung background dan beberapa menu.

    Fungsi ini juga bisa menampilkan informasi FPS (Frames Per Second) dalam layarmu (tambahkan kode comment “//” dalam baris

    director->setDisplayStats(true);

    jika kamu tidak ingin menampilkan informasi tersebut). FPS biasanya diatur pada 1/60 detik —jika lebih lambat dari itu maka mata manusia bisa mendeteksi lag, akan tetapi jika lebih cepat itu tidak akan berguna, karena terlalu cepat untuk dilihat mata manusia . Terakhir, sebuah scene dibuat dan object director diperintah untuk menjalankannya (perhatikan pada baris

    Director::getInstance()->runWithScene(scene)

    seharusnya digunakan untuk menjalankan scene pada awal game, karena Cocos akan menjalankan scene dengan sendirinya setelah itu).

#ifndef __HELLOWORLD_SCENE_H__
#define __HELLOWORLD_SCENE_H__
 
#include "cocos2d.h"
 
class HelloWorld : public cocos2d::Layer
{
public:
    static cocos2d::Scene* createScene();
 
    virtual bool init();
 
    // a selector callback
    void menuCloseCallback(cocos2d::Ref* pSender);
 
    // implement the "static create()" method manually
    CREATE_FUNC(HelloWorld);
};
 
#endif // __HELLOWORLD_SCENE_H__</pre>
Ini adalah file header dari class HelloWorld yang didefinisikan dalam HelloWorldScene.cpp. Tujuannya adalah untuk membuat sebuah scene untuk game milikmu. Di dalamnya mengandung deklarasi beberapa fungsi antara lain :
<ul>
    <li><strong><span style="color: #339966;">static</span> cocos2d::Scene* createScene()</strong> — Membuat sebuah scene (dengan satu buah layer) dan mengembalikan nilai pointer dari dirinya.
<ul>
    <li><em>Layers</em> — “Lembaran-lembaran” di dalam sebuah scene yang biasanya mengandung object. Setiap layer memiliki tingkat kedalaman (depth) (lihat di AppDelegate.cpp untuk definisi) yang dapat mempengaruhi perenderan gambar atau interaksi object.</li>
</ul>
</li>
    <li><strong><span style="color: #339966;">virtual</span> <span style="color: #3366ff;">bool</span> init()</strong> — Menginisialisasi scene HelloWorld untuk memiliki sebuah layer, sebuah tombol keluar (power) didalam aplikasi, Teks “Hello World” dan logo Cocos 2d</li>
    <li><strong><span style="color: #3366ff;">void</span> menuCloseCallback(cocos2d::Ref* pSender)</strong> — Dipanggil ketika user memencet tombol keluar (icon power) yang ada dalam aplikasi yang berfungsi untuk menutup program.</li>
    <li><strong>CREATE_FUNC(HelloWorld)</strong> — Adalah sebuah macro yang mendefinisikan method create() untuk classmu. Code sebenarnya dari macro tersebut adalah:
 
#define CREATE_FUNC(__TYPE__)
static __TYPE__* create()
{
    __TYPE__ *pRet = new(std::nothrow) __TYPE__();
    if (pRet && pRet->init())
    {
        pRet->autorelease();
        return pRet;
    }
    else
    {
        delete pRet;
        pRet = NULL;
        return NULL;
    }
}

Pada dasarnya, macro ini membuat sebuah instance HelloWorld(__TYPE__), yang dipanggil di fungsi init() (perhatikan, tanpa macro ini, fungsi init tidak akan pernah dipanggil!), dan panggil fungsi autorelease().

  • __TYPE__ — Ini adalah nama dari class yang macro gunakan (dalam hal ini, HelloWorld)
  • std::nothrow — Maksudnya bahwa jika fungsi tersebut gagal untuk membuat sebuah object baru dari type __TYPE__, maka itu harus mengembalikan null pointer daripada throwing an exception.
  • autorelease() — Walaupun kita membuat object baru menggunakan keyword "new", kita tidak pernah ingin untuk menghapusnya! Itu karena Cocos 2d menggunakan reference counters untuk melacak kapan untuk me-release (hapus) object.

#include "HelloWorldScene.h"
#include "SimpleAudioEngine.h"
 
USING_NS_CC;
 
Scene* HelloWorld::createScene()
{
// 'scene' is an autorelease object
auto scene = Scene::create();
 
// 'layer' is an autorelease object
auto layer = HelloWorld::create();
 
// add layer as a child to scene
scene->addChild(layer);
 
// return the scene
return scene;
}
 
// on "init" you need to initialize your instance
bool HelloWorld::init()
{
//////////////////////////////
// 1. super init first
if ( !Layer::init() )
{
return false;
}
 
auto visibleSize = Director::getInstance()->getVisibleSize();
Vec2 origin = Director::getInstance()->getVisibleOrigin();
 
/////////////////////////////
// 2. add a menu item with "X" image, which is clicked to quit the program
// you may modify it.
 
// add a "close" icon to exit the progress. it's an autorelease object
auto closeItem = MenuItemImage::create(
"CloseNormal.png",
"CloseSelected.png",
CC_CALLBACK_1(HelloWorld::menuCloseCallback, this));
 
closeItem->setPosition(Vec2(origin.x + visibleSize.width - closeItem->getContentSize().width/2 ,
origin.y + closeItem->getContentSize().height/2));
 
// create menu, it's an autorelease object
auto menu = Menu::create(closeItem, NULL);
menu->setPosition(Vec2::ZERO);
this->addChild(menu, 1);
 
/////////////////////////////
// 3. add your codes below...
 
// add a label shows "Hello World"
// create and initialize a label
 
auto label = Label::createWithTTF("Hello World", "fonts/Marker Felt.ttf", 24);
 
// position the label on the center of the screen
label->setPosition(Vec2(origin.x + visibleSize.width/2,
origin.y + visibleSize.height - label->getContentSize().height));
 
// add the label as a child to this layer
this->addChild(label, 1);
 
// add "HelloWorld" splash screen"
auto sprite = Sprite::create("HelloWorld.png");
 
// position the sprite on the center of the screen
sprite->setPosition(Vec2(visibleSize.width/2 + origin.x, visibleSize.height/2 + origin.y));
 
// add the sprite as a child to this layer
this->addChild(sprite, 0);
 
return true;
}

{
//Close the cocos2d-x game scene and quit the application
Director::getInstance()->end();
 
#if (CC_TARGET_PLATFORM == CC_PLATFORM_IOS)
exit(0);
#endif

}

File ini mengimplementasi fungsi-fungsi yang dideklarasikan oleh HelloWorldScene.h.

  • Scene* HelloWorld::createScene() — Membuat sebuah scene dan sebuah layer (sebuah instance dari class HelloWorld, karena class HelloWorld turunan dari Layer) yang dimana telah diatur sebagai child dari scene itu. Kemudian mengembalikan nilai sebuah pointer ke scene itu.
  • bool HelloWorld::init() — HelloWorld adalah turunan (inheritance) dari Layer, jadi hal yang pertama yang fungsi ini panggil adalah fungsi init kepunyaan Layer. Jika ia tidak dapat bekerja, kita kembalikan nilai false karena kita tidak dapat menginisialisasi HelloWorld jika kita bahkan tidak dapat menginisialisasi Layer. Kemudian fungsi ini membuat variable yang menyimpan ukuran layar sistem koordinat asal layar (digunakan untuk mengatur posisi dari object - object yang ada dalam layar). Ia juga membuat sebuah tombol keluar (icon power) di dalam aplikasi yang kegunaannya agar pengguna bisa menutup aplikasi tersebut (perhatikan, ini bukan tombol merah yang ada tulisan "X" yang ada di pojok kanan jendela aplikasi) dan mengatur posisinya. Kemudian membuat sebuah menu yang mengandung tombol keluar sebagai child object dan mengatur posisinya di layar. Kemudian, fungsi ini membuat sebuah label “Hello World” (Label::createWithTTF(text, font, font-size)),  mengatur posisinya di layar, dan menambahkannya sebagai child dari object HelloWorld. Kemudian, hal yang sama berlaku untuk sebuah sprite yang mengandung logo Cocos 2d. Yang akhirnya, fungsi tersebut mengembalikan nilai true untuk menunjukkan telah berhasil menginisialisasi object HelloWorld.
    • Label — Sebuah teks yang bisa ditampilkan pada layar anda
    • Sprite — Sebuah gambar
  • void HelloWorld::menuCloseCallback(cocos2d::Ref* pSender) — Berguna untuk menutup program.
